# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

OrbitEngine is a full-stack SaaS platform (FastAPI + React) for managing internal processes in SMEs. It uses AI-assisted demand forecasting and supports multi-tenancy via organizations.

## Commands

### Backend (`backend/` directory)

```bash
uv run fastapi dev app/main.py                                                   # Dev server
uv run bash scripts/test.sh                                                      # All tests with coverage
uv run pytest tests/api/routes/test_users.py -v                                  # Single test file
uv run pytest tests/api/routes/test_users.py::test_get_users_superuser_me -v     # Single test function
uv run pytest -k "test_create" -v                                                # Tests matching pattern
uv run bash scripts/lint.sh                                                      # mypy + ruff
uv run bash scripts/format.sh                                                    # Format code
```

### Frontend (`frontend/` directory)

```bash
bun run dev                                              # Dev server
bun run build                                            # Production build
bun run lint                                             # Biome lint + auto-fix
bun run test                                             # Playwright E2E tests
bunx playwright test tests/login.spec.ts                 # Single test file
bunx playwright test --grep "create item"                # Tests matching pattern
bun run test:ui                                          # Tests with interactive UI
bun run generate-client                                  # Regenerate API client (backend must be running)
```

### Docker Compose (project root)

```bash
docker compose watch                                                              # Full stack with hot reload
docker compose exec backend alembic revision --autogenerate -m "description"     # Create migration
docker compose exec backend alembic upgrade head                                  # Apply migrations
docker compose exec backend alembic downgrade -1                                  # Rollback one migration
```

## Architecture

```
Frontend (React/TS) → Auto-generated OpenAPI client → Backend (FastAPI) → PostgreSQL
                                                                         → S3/MinIO (file storage)
                                                                         → Resend / Mailcatcher (email)
```

### Backend (`backend/app/`)

- `models.py` — all SQLModel database models AND Pydantic schemas in one file. Schema naming: `{Model}Create`, `{Model}Update`, `{Model}Public`, `{Model}sPublic`.
- `crud.py` — all CRUD operations. Functions take `session` as keyword-only arg.
- `api/routes/` — one file per domain (users, products, categories, customers, sales, inventory_movements, organizations, dashboard, roles).
- `api/deps.py` — `CurrentUser`, `SessionDep`, and other `Annotated` dependency aliases used across routes.
- `core/` — settings, security (JWT), and database engine setup.
- `alembic/` — database migrations; must be run inside Docker to reach the database.

### Frontend (`frontend/src/`)

- `client/` — **auto-generated from OpenAPI, do not edit manually**. Regenerate with `bun run generate-client` after backend changes.
- `components/ui/` — **shadcn/ui components, do not edit manually**.
- `routeTree.gen.ts` — **auto-generated by TanStack Router, do not edit manually**.
- `routes/` — file-based routing via TanStack Router; dashboard sub-routes live under `routes/dashboard/`.
- `hooks/` — custom hooks: `useAuth`, `useCopyToClipboard`, `useCustomToast`, `useMobile`.

## Code Style

### Backend (Python)

- Full type hints on all functions. Use `str | None` not `Optional[str]`.
- Use `Annotated` for dependency injection (see `deps.py` patterns).
- Route handlers return `Any`.
- Raise `HTTPException` with: 400 (bad request), 403 (forbidden), 404 (not found), 409 (conflict).
- Create DB objects with `model_validate()`; partial updates with `model_dump(exclude_unset=True)`.
- Always `session.commit()` + `session.refresh()` after mutations.
- Import order: stdlib → third-party → local (`app.*`).

### Frontend (TypeScript/React)

- Functional components only.
- React Hook Form + Zod for all forms.
- TanStack Query for server state; always `invalidateQueries` after mutations.
- Internal imports use `@/` alias.
- Biome enforces: spaces, double quotes, semicolons only as needed.

## Environment Setup

Copy `.env.example` to `.env` and fill in: domain, PostgreSQL credentials, `SECRET_KEY`, Resend API key, and S3/MinIO credentials. Docker Compose provides PostgreSQL, MinIO, Mailcatcher, Adminer, and Traefik locally.

## Testing Notes

Backend test fixtures (`conftest.py`): `db`, `client`, `superuser_token_headers`, `normal_user_token_headers`. Mock external services with `unittest.mock.patch`. Frontend E2E auth state is managed via `tests/utils/auth.setup.ts`.
